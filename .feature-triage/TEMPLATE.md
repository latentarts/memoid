# Feature Triage Template

## Metadata

- Status: `draft`
- GitHub Issue:
- Owner:
- Last Updated:

## Title

Short feature name.

## Problem

What problem exists today and who experiences it.

## Proposed Behavior

What the system should do if this feature is implemented.

## Rationale

Why this is worth doing now or eventually.

## Scope

What is in scope and what is intentionally out of scope.

## Risks

Known risks, ambiguities, or tradeoffs.

## Notes

Optional implementation notes, examples, or supporting context.

## Promotion Checklist

- [ ] The note is mature enough to become a trackable issue.
- [ ] The note has a clear problem statement and expected behavior.
- [ ] The note is marked `ready-for-issue`.
- [ ] A duplicate issue search has been performed.
- [ ] The created GitHub issue number or URL has been written back here.
